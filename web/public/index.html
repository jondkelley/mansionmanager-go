Unregistered palaces

Not in registry.json anymore (e.g. after Unregister only). Palace Manager keeps a snapshot in /etc/palace-manager/unregistered-palaces.json so they stay listed here even if systemd does not show the unit. Use Register… to put them back in the registry and restore nginx routing; restore or recreate the palman- unit on the host if you need the service to start.

Update pserver Binary

Downloads the latest official pserver tarball, refreshes the template at /root/palace-template, and updates /usr/local/bin/pserver.

First-time setup: run Update Binary before creating any Palace instances — it downloads the pserver template that provisioning depends on.

Rollouts & pinned binaries

Each successful Update Binary archives pserver by semver from version.txt under the versions directory (below). Choose latest to use the shared install at /usr/local/bin/pserver, or pin a palace to an archived build. Applying patches the systemd unit and updates the registry; optionally restart so the service loads the binary immediately.

Media proxy & TLS edge

These settings mirror /etc/palace-manager/config.json (nginx.edgeScheme, matchScheme, etc.). The computed URL is passed to each pserver as --reverseproxymedia. Changing between http and https rewrites every palman-*.service unit; restart palaces so running processes publish the new public media URL (same as toggling TLS on the nginx edge).
