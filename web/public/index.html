Update pserver Binary

Downloads the latest official pserver tarball, refreshes the template at /root/palace-template, and updates /usr/local/bin/pserver.

First-time setup: run Update Binary before creating any Palace instances — it downloads the pserver template that provisioning depends on.

Rollouts & pinned binaries

Each successful Update Binary archives pserver by semver from version.txt under the versions directory (below). Choose latest to use the shared install at /usr/local/bin/pserver, or pin a palace to an archived build. Applying patches the systemd unit and updates the registry; optionally restart so the service loads the binary immediately.
