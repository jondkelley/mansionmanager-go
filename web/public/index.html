Use Register… to put them back in the registry and restore nginx routing; restore or recreate the palman- unit on the host if you need the service to start.

Version pins

New palaces are automatically pinned to the pserver version that was current when they were created. Set a palace to latest to always pick up the newest binary on restart — this is opt-in. Each Pull Latest archives the new build so you can roll back to any earlier version.

Media proxy & TLS edge

Mirrors /etc/palace-manager/config.json (nginx.*). The computed URL is passed to each pserver as --reverseproxymedia. Changing the edge scheme rewrites every palman-*.service unit — restart palaces so running processes publish the new URL.

Host Setup

Run once on a fresh host to install system dependencies, obtain a Let\'s Encrypt certificate, and generate the initial nginx config. Each step is idempotent — safe to re-run. Uses the Media hostname and Edge scheme saved above.
